# Día 09: Proyectos Integradores (Evaluación Final)

Esta sesión constituye la evaluación final del módulo. Durante esta etapa, no se impartirán clases magistrales teóricas ni se desarrollará código guiado. El objetivo principal es que el estudiante demuestre el dominio de los conocimientos adquiridos mediante su aplicación en escenarios de negocio reales.

## Instrucciones para la Evaluación

El estudiante debe seleccionar (o ser asignado por el equipo docente a) **UNO** de los siguientes cuatro mega-proyectos. Cada cuaderno proporcionado funciona como una plantilla estructurada que guiará al estudiante a través de cinco fases críticas de desarrollo:
1. **Comprensión de Datos.**
2. **Data Munging (Limpieza y Transformación).**
3. **Análisis Exploratorio (EDA).**
4. **Comparativa Algorítmica.**
5. **Defensa Analítica y Ética de Negocios.**

## Los Desafíos

1.  **`09_Proyecto_1_NLP_Sentimiento.ipynb`:** Se requiere analizar reseñas de clientes provenientes de Amazon utilizando técnicas de Procesamiento de Lenguaje Natural (NLP), con el objetivo de prevenir crisis de relaciones públicas.
2.  **`09_Proyecto_2_Clustering_Retail.ipynb`:** Se debe aplicar aprendizaje automático no supervisado (K-Means) para descubrir y segmentar los perfiles comerciales ocultos dentro de la base de clientes de un Centro Comercial.
3.  **`09_Proyecto_3_Riesgo_Salud.ipynb`:** Se evaluarán historiales médicos para predecir el riesgo de infartos, requiriendo que el estudiante justifique éticamente el impacto de los errores de predicción en el ámbito clínico.
4.  **`09_Proyecto_4_Demanda_Transporte.ipynb`:** Se optimizará la logística de una ciudad inteligente (Smart City) mediante el desarrollo de modelos predictivos que calculen la demanda de transporte urbano en intervalos horarios.

**¡Mucha suerte a todos! El límite es la imaginación y el rigor analítico.**
